Ingestion
Kafka Data Sources
🖮 IOS-XR Router Telemetry
2 routers → pipeline-gnmi → router_metrics
gNMI subscription streams interface counters, CPU, memory, and BGP state from live IOS-XR routers.
📈 Synthetic Traffic Generator
datagen → app_metrics
Simulates application latency, throughput, and error rates across systems like System Thor and branch sites.
🔒 Security Events
datagen → security_events
Firewall denies, IPS alerts, and authentication failures correlated with network anomalies.
☁️ Cloud Infra Events
datagen → cloud_events
Cloud VM health, autoscaling events, and service degradation signals from hybrid infrastructure.
02
Stream Processing
Flink SQL joins, correlates, and detects anomalies in real time
Flink SQL
Real-Time Stream Analytics

⚙️ Processing Pipeline

  • Join all streams — correlate router, app, security, and cloud events by site and timestamp
  • Detect anomalies — flag spikes in latency, packet loss, or security denials
  • Create health score — composite Green / Yellow / Red per branch
  • Generate incidents — open structured incident records on threshold breach
  • Generate RCA events — enrich incidents with correlated evidence for AI analysis

📦 Kafka Output Topics

  • health_status — rolling health scores per site
  • incidents — active and resolved incident records
  • rca_events — evidence bundles for LLM root cause analysis
-- Flink SQL: join streams by site
SELECT r.site, r.if_util, a.latency_ms,
  s.alert_count, c.vm_status
FROM router_metrics r
JOIN app_metrics a ON r.site = a.site
JOIN security_events s ON r.site = s.site
JOIN cloud_events c ON r.site = c.site
